## Critical (fix now)

**C1 · In-app photo import is structurally broken — full client↔fn contract mismatch.**
- index.html:1464 sends `{image: b64, mediaType}`; fn (parse-workout/index.ts:52-55) reads `body.images` → `[]` → 400 "no images". This IS the live "Parse failed: non-2xx".
- Even with the key fixed, response contract diverges: client (1469-1475) reads `data.pace/hr/stations/notes/run_splits` — fn returns `avg_pace/avg_hr` and never returns stations/notes/run_splits.
- Design divergence: fn **inserts the session itself** (service-role, needs_review) for the Shortcut flow; client expects parse-and-prefill for manual save → if request were fixed as-is, one screenshot = fn insert + user save = **double entry**.
- Also: fn gates on `x-hachi-key` when set; `sb.functions.invoke` never sends it → 401 whenever the secret exists.
- Fix: pick one contract. Recommended: fn accepts `{images:[…], insert:false}` mode returning parse-only with client's field names; Shortcut keeps `insert:true`. One fn, two modes.

**C2 · parse-workout auth is fail-open.** index.ts:47 `if (ingestKey && …)` — if `HACHI_INGEST_KEY` env is unset/deleted, the endpoint is unauthenticated with service-role insert behind it. Combined with known state (key still `hachi-test-123`): weak-or-absent gate on a write path. Fix: hard-fail when env missing; rotate key to prod value.

**C3 · `hyresult` Edge fn source not in repo.** index.html:1723 invokes it; `supabase/functions/` contains only parse-workout. Deployed-only code = two-sources-of-truth trap #1 (already bit once with PNG/dedupe regression). Also the split-import bug lives there and can't be audited/fixed from repo. Fix: pull deployed source, commit, then fix splits.

## High (fix this sprint)

**H1 · seedWeekSessions destructive replace.** :1883-1905 `S.sessions = seeded` whenever `<14` sessions — on a degraded load (sessions select error on fresh device: loadFromSupabase:1972 `if(!se.error)` leaves `S.sessions=[]`), seeds replace state and pushAll INSERTs 14 stale rows (string ids = "fresh"). Bootstrap long done. Fix: delete the function.

**H2 · Race-import benchmarks upsert missing onConflict.** :1732 `upsert(bm)` vs :1988 `upsert(bm,{onConflict:'user_id,station'})` — without it, conflicts 409 (error unchecked, silently swallowed) or duplicate rows. Fix: add onConflict + check error.

**H3 · UTC date defaults — the documented `_localISO` lesson violated in 4 places.** :1308 (HRV), :1483 + :1948 (log modal, duplicated logic), :1726 (race import fallback). Before 09:00 JST all default to *yesterday*. lessons.md explicitly names log/HRV defaults. Fix: `_localISO()` everywhere; delete the :1948 duplicate.

**H4 · Fn-side UTC + year-clamp corrupts dates.** index.ts:93-99: dateless screenshot → `toISOString().slice(0,10)` (UTC = yesterday pre-09:00 JST, server-side this time); any past-year date force-rewritten to current year (a Dec 2025 screenshot parsed in Jan 2026 becomes **future** 2026-12-xx). Fix: JST-aware date; clamp only when resulting date ≤ today.

**H5 · listUsers() per request.** index.ts:118-120 lists all auth users to resolve one email — breaks silently past the default page size (50) as the group grows; slow; admin surface on every ingest. Fix: `profiles` lookup by email or `getUserByEmail` equivalent.

**H6 · pushAll insert errors discarded.** :1984 `const {data}` — no error check; failed inserts (RLS/network) silently retry forever as "fresh" rows; user never told sync failed. Fix: surface error once per session; cap retries.
